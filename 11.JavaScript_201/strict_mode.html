Strict Mode

strict mode in JavaScript can be enabled by adding "use strict"; to your code.

Strict mode will be much less tolerant and throw errors if you write ambiguous statements. For example:

"use strict";
const x = 1;  // no error
let y = 1;  // no error
var z = 1;  // no error
a = 1;  // 'ReferenceError: a is not defined'

Without strict mode enabled, a would be declared as a global variable as if var had been used.

From MDN:

Strict mode changes some previously-accepted mistakes into errors. JavaScript was designed to be easy for novice developers, and sometimes it gives operations which should be errors non-error semantics. Sometimes this fixes the immediate problem, but sometimes this creates worse problems in the future. Strict mode treats these mistakes as errors so that they're discovered and promptly fixed.
